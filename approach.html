The Approach

As a web developer, responsive design is designing your website so that it responds to your users' environment making it bulletproof in all condition of screen sizes. Responsive design is not a design for mobile or tablet, it is a design for ‘____’ (blank, which means the page has to fit in any type of environment).

CSS Relative Units

CSS Relative Units is one of the key tips for responsive design to allow the flexibility in sizing text, image or length and width of an element in general.

Absolute vs. Relative lengths

Absolute lenghts is the absolute length units are fixed and a length expressed in any of these will appear as exactly that size. Absolute length units are not recommended for use on screen, because screen sizes vary so much. However, they can be used if the output medium is known, such as for print layout. Example: , cm, mm, pt

Relative lengths is Relative length units specify a length relative to another length property. Relative length units scales better between different rendering mediums. Example: em, ex, ch, rem

Responsive Image

Responsive image is one of the most important part of responsive web design. Image in responsive website needs to be in relative units. Having the sizes in the relative units would prevent the image from placing outide their containing element.

Common Approach

The common approach for reposive image is using the width or max-width property

img { width: 100%; height: auto;}

or

img { max-width: 100%; height: auto;}

The Viewport

Viewport is the user's visible area of a web page. The viewport is scalable, it will be smaller on a mobile phone than on a computer screen. When the web was designed only for computer screens, and it was common for web pages to have a static design and a fixed size. Then, when we started surfing the internet using tablets and mobile phones, fixed size web pages were too large to fit the viewport. To fix this, browsers on those devices scaled down the entire web page to fit the screen. This was not perfect, but a quick fix.

CSS Media Queries

A media query allows us to target not only certain device classes, but to actually inspect the physical characteristics of the device rendering our work. The @media rules are used to define different style rules for different media types/devices. Media queries allow you to set the CSS to react differently at different pixel width of the screen.

@media (query) { /* CSS Rules used when query matches */}

The query contains two components

  1. A media type (screen)
  2. the actual query enclosed within parentheses, containing a particular media feature (max-device-width) to inspect, followed by the target value (480px)

Example Code

@media screen and (max-device-width: 480px) {.column {float: none;} }

Responsive Framework

Since the day that Responsive Web Design becomes the main stream and requirement for most modern website, there are different of responsive web design patterns and frameworks were created.

Responsive Web Design Patterns

Most layouts used by responsive web pages can be categorized into one of five patterns: mostly fluid, column drop, layout shifter, tiny tweaks, and off canvas. In some cases, a page may use a combination of patterns, for example column drop and off canvas.

Example

Mostly Fluid: The mostly fluid pattern consists primarily of a fluid grid. On large or medium screens, it usually remains the same size, simply adjusting the margins on wider screens. On smaller screens, the fluid grid causes the main content to reflow, while columns are stacked vertically. One major advantage of this pattern is that it usually only requires one breakpoint between small screens and large screens.

In the smallest view, each content div is stacked vertically. When the screen width hits 600px, the primary content div remains at width: 100%, while the secondary div's are shown as two columns below the primary div. Beyond 800px, the container div becomes fixed width and is centered on the screen.

Responsive Web Design Frameworks

Similar to Patterns, there are different frameworks that you can find online such as Bootstrap, Foundation, Skeleton, etc. All of them share a similar thing, the grid system.

Grid System

The most popular responsive website framework - Bootstrap uses the 12 Divided Columns, Grid System

Using a grid-view is very helpful when designing web pages. It makes it easier to place elements on the page. A responsive grid-view often has 12 columns, and has a total width of 100%, and will shrink and expand as you resize the browser window.
